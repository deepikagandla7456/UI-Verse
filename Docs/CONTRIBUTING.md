You can contribute in several ways:

- Report bugs by submitting reproducible issue reports.
- Suggest features or improvements.
- Improve documentation for clarity, grammar, or structure.
- Add code to fix bugs, build new features, or optimize existing ones.
- Test the project in different environments.

## Getting Started

1. Fork the repository.
2. Clone your fork.
3. Create a branch.
4. Make your changes.
5. Commit your changes.
6. Push your branch.
7. Create a pull request.

Example commands:

```bash
git clone https://github.com/your-username/uiverse.git
git checkout -b feature/your-feature-name
git commit -m "Added: new feature"
git push origin feature/your-feature-name
```

## PR Review Process

We aim to keep things smooth and transparent:

- Once your PR is submitted, a maintainer will review it.
- You may be asked to:
  - Fix styling issues.
  - Add missing documentation or tests.
  - Break large PRs into smaller pieces.
- After approval:
  - Your PR will be merged with a squash merge to keep history clean.
  - You will receive feedback, even if the PR is not merged immediately.
- Reviews may take **24–72 hours** depending on activity. Thanks for your patience.

For any query, send a DM on LinkedIn: [Tushar Sonawane](https://www.linkedin.com/in/tushar-sonawane-800422336/).

## How To Contribute

- Drop a star in this repo.
- Review the existing [issues](https://github.com/Tushar-sonawane06/UI-Verse/issues).
- Fork the repo and create a branch for the issue you are working on.
- Open an issue before starting work.
- Use the GitHub issue template for bug reports or feature requests so maintainers can triage them consistently.
- Wait to be assigned before opening a PR.
- Create a [pull request](https://github.com/Tushar-sonawane06/UI-Verse/pulls) after assignment.
- Add screenshots or screen captures to help explain the changes.
